Safety and Security

Data Privacy and Protection for Metroshuttle App

Metroshuttle is dedicated to safeguarding the privacy and security of our users' personal information. We are committed to transparent data handling practices and comply with all applicable data protection laws.

Data Collected and Its Purpose
1. User Information: Name, contact details, payment information. Used for booking, payment processing, and customer support.
2. Location Data: Real-time location data for matching users with available shuttles. This data is essential for the app's core functionality.
3. Ride History: Records of past rides for user convenience and service improvement.
4. Device Information: Device type and operating system for app optimization and troubleshooting.

Data Security Measures
1. Encryption: Sensitive data, such as payment information, is encrypted both in transit and at rest.
2. Access Controls: Strict access controls limit data access to authorized personnel.
3. Regular Security Audits: We conduct regular security assessments to identify and address potential vulnerabilities.
4. Data Minimization: We only collect and retain the data necessary for our services.

User Rights

Users have right to

  • Access their personal data
  • Correct inaccurate data
  • Request data deletion
  • Object to data processing
  • Restrict data processing

Data Sharing

We do not share personal information with third parties, except when required by law or with explicit user consent. For example, we may share aggregated, anonymized data with partners for service improvement purposes.

Data Retention

We retain user data for as long as necessary to provide our service or as required by law.

Background Checks for Drivers

To ensure passenger safety, Metroshuttle conducts thorough background checks on all drivers, including:

  • Criminal record checks
  • Driving record checks
  • Reference checks

We retain background check information and conduct periodic re-checks to maintain driver eligibility.

Emergency Procedures

Metroshuttle has established emergency procedures to handle unforeseen incidents:

  • Real-time tracking: Continuous monitoring of shuttle locations for immediate response.
  • Emergency contacts: Users are encouraged to provide emergency contact information.
  • Driver training: Drivers undergo training on emergency procedures and first aid.
  • Collaboration with local authorities: We maintain close cooperation with local emergency services.

By using the Metroshuttle app, you agree to our data privacy policy, driver screening process, and emergency procedures.

Technology Stack

Metroshuttle was developed using Flutter that offers a robust platform for cross-platform development, and combining it with Google Maps API and Firebase.
Core Technologies.
Flutter: As the primary framework, Flutter handles the app's UI, logic, and state management across iOS and Android platforms.
Google Maps API: Provides mapping functionalities, including displaying routes, markers, and real-time location tracking.
Firebase: Offers a comprehensive suite of services, likely including:
>Firebase Realtime Database: For storing and synchronizing app data in real-time
>Firebase Cloud Messaging(FCM): For push notifications to inform users about ride status, updates, and promotions.
>Firebase Authentication: For user authentication and authorization.
>Firestore: For storing images or other media.
While Flutter, Google Maps API, and Firebase cover the core functionalities of Metroshuttle, Geolocation is also used to determine user's location for ride requests and matching with nearby vehicles.
